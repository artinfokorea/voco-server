# 공통 명세

## 응답 형식

### 성공 응답
```json
{
  "type": "SUCCESS",
  "exception": null,
  "item": { ... }
}
```

### 실패 응답
```json
{
  "type": "FAIL",
  "exception": {
    "errorNo": "에러번호",
    "message": "에러메시지"
  },
  "item": null
}
```

---

## 에러 코드 정의

### 시스템 에러
| 에러 번호 | 메시지 |
|-----------|--------|
| SYSTEM-1 | 서버에서 오류가 발생했습니다. 요청을 처리할 수 없습니다. |
| SYSTEM-2 | 유효하지 않은 요청입니다. |

### 인증 에러
| 에러 번호 | 메시지 |
|-----------|--------|
| AUTH-1 | 토큰이 존재하지 않습니다. |
| AUTH-2 | 토큰이 만료되었습니다. |
| AUTH-3 | 토큰 서명이 유효하지 않습니다. |
| AUTH-4 | 잘못된 형식의 토큰입니다. |
| AUTH-5 | 지원하지 않는 토큰입니다. |
| AUTH-6 | 유효하지 않은 토큰입니다. |

### 회원 에러
| 에러 번호 | 메시지 |
|-----------|--------|
| MEMBER-1 | 해당 유저가 존재하지 않습니다. |
| MEMBER-2 | 비밀번호는 최소 8자 이상이며, 영문자, 숫자, 특수문자를 각각 하나 이상 포함해야 합니다. |
| MEMBER-3 | 비밀번호가 일치하지 않습니다. |
| MEMBER-4 | 이미 사용 중인 이메일입니다. |
